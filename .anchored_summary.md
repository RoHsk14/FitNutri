## Goal
- Build a complete Fitness & Nutrition web app (Next.js + Supabase) where users onboard with photos and a goal description, get personalized training/nutrition plans with estimated duration, log daily meals/weight/water, and follow a gender-aware workout program with exercise visuals and filters.

## Constraints & Preferences
- Stack: Next.js 14 App Router, Supabase (PostgreSQL + Auth + Storage), TypeScript, Tailwind CSS, Zod, Recharts
- Shared Supabase project: tables prefixed with `fit_`, storage bucket `fit-physique-images`
- No real auth yet: temporary cookie-based user ID (`fit_user_id`); `clerk_user_id` placeholder in schema
- Server actions use `SUPABASE_SERVICE_ROLE_KEY` (in `.env`, gitignored) to bypass RLS
- Image upload to public Supabase Storage bucket (5MB limit per file)
- Duration estimator has NO caps: output weeks ~2–156, varies with gender/goal/weight diff/activity/sessions/meals/age/BMI
- Gemini API free tier (Google AI Studio, no credit card, 1500 req/day) for physique image analysis during onboarding
- Exercise data sourced from Muscle & Strength (352 exercises, all with video URLs, classified by muscle/mechanics/equipment)

## Key Decisions
- 352 M&S exercises scraped from 12 muscle category pages + 25 manually added core exercises (barbell-bench-press, push-up, squat, etc. missing from M&S category tags)
- Equipment inferred from exercise name (barbell/dumbbell/cable keywords) + manual overrides for 40+ exercises
- Muscle overrides fix M&S miscategorizations (e.g., bent-over-barbell-row → Dos, not Abdominaux)
- French names map 61 exercises to match our app's French UI; remaining 291 keep English names
- The existing filter/selector system works as-is with the new taxonomy data
